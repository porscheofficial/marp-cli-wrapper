## Reporting an Issue

Found a bug or have an idea for improvement?

1. Check if your issue has already been reported: [See Issues](https://github.com/porschedigital/marp-cli-wrapper/issues)
2. Create a new issue with a clear description: [Create Issue](https://github.com/porschedigital/marp-cli-wrapper/issues/new)

Please provide:

- A clear description of the issue
- Steps to reproduce (if it's a bug)
- Expected vs actual behavior
- Your environment (OS, Node version, etc.)

Want to contribute code? That's awesome! ğŸ‰

### Getting Started

1. **Checkout the repository**

2. **Install dependencies**

```bash
pnpm install
```

3. **Create a feature branch**

```bash
git switch -C feature/your-feature-name
```

### Development Workflow

1. **Make your changes** following the existing code style
2. **Run tests locally**

```bash
pnpm run test
```

3. **Run type checking**

```bash
pnpm run type-check
```

4. **Run security audit**

```bash
pnpm run audit
```

### Commit Messages

We use **Conventional Commits** for clear and consistent commit history. See [https://www.conventionalcommits.org/](https://www.conventionalcommits.org/)

### Before Submitting

1. **Push your branch**

```bash
git push origin feature/your-feature-name
```

2. **Create a Pull Request**
   - Use a clear title following Conventional Commits
   - Reference any related issues
   - Describe what your PR changes and why
